# Security Policy

<!-- Implements: ENTERPRISE-READINESS.md → "Security incident response" -->

## Reporting a vulnerability

If you discover a security vulnerability in this project, please report it responsibly.

**Email:** security@cityfriends.tech

**Do not:**

- Open a public GitHub issue
- Post about the vulnerability on social media
- Exploit the vulnerability beyond what's necessary to demonstrate it

## Response timeline

- **72 hours:** Acknowledgment of your report
- **7 days:** Initial assessment and severity determination
- **30 days:** Target for fix deployment (varies by severity)

You will be kept informed of progress throughout.

## Scope

### In scope

- The production application and APIs
- Authentication and authorization systems
- Data handling and storage
- Third-party integrations we control

### Out of scope

- Third-party services we don't control (report to them directly)
- Social engineering attacks against team members
- Physical security
- Denial of service attacks

## Safe harbor

We consider security research conducted in good faith to be authorized. We will not pursue legal action against researchers who:

- Make a good faith effort to avoid privacy violations, data destruction, or service disruption
- Report vulnerabilities to us before disclosing publicly
- Give us reasonable time to address the issue before disclosure

## Recognition

We're happy to acknowledge security researchers who help us improve. Let us know if you'd like to be credited in release notes or our security hall of fame.
