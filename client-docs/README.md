# Client Documents

This folder holds materials provided by the client or government agency at the start of the project — briefs, prior research, sample data, evaluation criteria, technical specifications, design assets.

## Folder structure

```
client-docs/
├── raw/                Original materials, untouched
└── synthesized/        Output from research synthesis (typically Qori)
```

## raw/

Materials exactly as the client provided them. PDFs, Word docs, spreadsheets, original filenames preserved. Don't edit, rename, or reformat. They're the source of truth.

## synthesized/

Processed versions — typically the output of running raw materials through Qori (the Friends Innovation Lab research synthesis tool). Qori turns multiple research files into structured documentation the team can work from.

The Qori workflow:

1. Upload raw materials to Qori through Slack
2. Qori processes the documents and writes structured output to the `qori-studies` repo
3. Download the relevant synthesis files from `qori-studies` and add them here

For the full operational workflow, see [`02-running-a-project/00-when-a-proposal-lands.md`](https://github.com/friends-innovation-lab/playbook/blob/main/02-running-a-project/00-when-a-proposal-lands.md) in the playbook.

## What does NOT belong here

- Code (use the project's source folders)
- Project planning artifacts like PRD, epics, ADRs (use `/docs/`)
- Internal team notes (use `/docs/notes/` if needed)
- Anything generated by the team during the build (use `/docs/`)

This folder is specifically for inputs from outside the lab — the materials we receive that inform what we build, not the materials we produce while building it.
